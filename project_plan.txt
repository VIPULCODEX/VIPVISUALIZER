--------------------------------------------------------------------------------
BACKGROUND & MOTIVATION
--------------------------------------------------------------------------------

Kidney Exchange Programs (KEP) match incompatible donor-recipient pairs so that
compatible swaps can be made. The standard approach models this as finding cycles
(2-way, 3-way swaps) and chains (non-directed donor-initiated) in a compatibility
graph and solves it using Integer Linear Programming (ILP).

The problem with current KEP solvers:
  - Cycles and chains are optimized independently.
  - Real exchanges can CONFLICT: two exchange sets may be medically linked through
    shared crossmatch tests, tissue type overlaps, or hospital logistics. Activating
    both causes cascade failures.
  - Current ILP models completely ignore these conflict relationships.

This project introduces a new model and a new algorithm that addresses this gap.

PAPER BACKGROUND (Papers Already Read)
--------------------------------------------------------------------------------

1. "The Parameterized Complexity of the Induced Matching Problem"
   - Authors: Hannes Moser and Somnath Sikdar
   - Key Results:
       * Induced matching is NP-complete and W[1]-hard on general graphs.
       * Linear kernel on planar graphs (via 3 reduction rules + maximal region decomposition).
       * Linear kernel on bounded-degree graphs.
       * Cubic kernel (O(k^3)) on graphs with girth >= 6.
       * FPT on line graphs.
       * Linear-time DP on bounded treewidth graphs (4^treewidth factor).
       * W[1]-hard on bipartite graphs (via reduction from Irredundant Set).

2. Additional papers on acyclic matching and kidney exchange (read by student).


--------------------------------------------------------------------------------
THE NEW PROBLEM: Mixed Induced-Acyclic Matching (MIAM)
--------------------------------------------------------------------------------

DEFINITIONS:

  Compatibility Graph G = (V, E):
    - Each vertex v in V = a donor-recipient pair
    - Edge (u, v) in E = donor of u is compatible with recipient of v, or vice versa

  Conflict Graph C = (V, F):
    - Same vertex set V
    - Edge (u, v) in F = pairs u and v cannot both be activated (conflict exists due to
      shared crossmatch, hospital constraint, tissue type overlap, etc.)

  Induced Matching (for cycle exchanges):
    - A matching M in G such that no two edges of M are connected by any other edge.
    - Ensures activated cycle-exchanges are fully independent of each other.

  Acyclic Matching (for chain exchanges):
    - A matching M in G such that the subgraph induced by M contains no alternating cycles.
    - Models chain-based exchanges initiated by non-directed (altruistic) donors.

  Mixed Induced-Acyclic Matching (MIAM):
    - A set S of exchanges (edges/cycles/chains) in G such that:
        (1) S forms a valid matching (no vertex used twice).
        (2) No two elements of S are connected by a conflict edge in F (induced constraint).
        (3) The chain-portion of S forms an acyclic matching (no alternating cycles).
    - Goal: Find maximum weight MIAM of size >= k.

WHY THIS IS NEW:
  - No existing paper unifies induced matching + acyclic matching in a single framework.
  - No existing work applies FPT techniques to weighted KEP under conflict constraints.
  - No paper exploits blood-type/HLA biological structure for graph algorithm design.

RESEARCH CONTRIBUTIONS (What We Will Prove and Build)
--------------------------------------------------------------------------------

CONTRIBUTION 1 — Structural Observation on Kidney Exchange Graphs
  - Prove that real kidney exchange compatibility graphs are structurally restricted:
      * Blood type compatibility (O, A, B, AB) limits expected degree to <= 6.
      * HLA crossmatch further restricts the graph to near-planar or bounded-degree.
  - Formally characterize the graph class that kidney exchange graphs belong to.
  - This justifies applying the linear kernel from Moser-Sikdar to real KEP instances.

CONTRIBUTION 2 — Weighted Kernelization (New Reduction Rules)
  - Existing kernels only work for UNWEIGHTED induced matching.
  - We extend to WEIGHTED case (patient urgency, waiting time, pediatric priority).
  - Design 4-5 new reduction rules that preserve weights:
      Rule 1: Remove isolated vertices (degree 0 in both G and C).
      Rule 2: Remove dominated pairs (a vertex strictly worse in weight and compatibility).
      Rule 3: Forced inclusion (a vertex with no conflict neighbors and high weight).
      Rule 4: Conflict chain simplification (simplify long conflict chains).
      Rule 5: [To be derived based on kidney graph structure].
  - Prove: After applying rules, kernel size is O(k) for bounded-degree kidney graphs.

CONTRIBUTION 3 — W[1]-Hardness on General Graphs
  - Prove MIAM is W[1]-hard on general bipartite graphs.
  - Reduction from W[1]-complete Induced Matching on bipartite graphs
    (already proven hard by Moser-Sikdar via Irredundant Set reduction).
  - This shows that restricting to kidney graph structure is NECESSARY for tractability.

CONTRIBUTION 4 — FPT Algorithm for MIAM on Kidney Graphs
  - Combine kernelization + bounded search tree + dynamic programming.
  - Algorithm sketch:
      Step 1: Apply weighted reduction rules to get kernel of size O(k).
      Step 2: Run bounded search tree on the kernel.
      Step 3: For each branch, apply treewidth-based DP (O(4^tw * n)).
  - Target complexity: O(f(k) * n) where f(k) is a computable function of k.
  - Prove correctness and complexity bound formally.

CONTRIBUTION 5 — Implementation and Benchmarking
  - Implement the full algorithm pipeline.
  - Build synthetic kidney exchange graph generator (using real blood type distributions).
  - Benchmark against:
      * Gurobi/GLPK ILP (standard KEP baseline)
      * Greedy matching
      * Brute force (small instances for correctness check)
  - Datasets:
      * Synthetic graphs (100 to 10,000 vertices)
      * Real datasets: UNOS kidney exchange data, UK NHS KEP public data (if accessible)
  - Metrics: Solution quality (total weight), runtime, scalability with k, kernel reduction ratio.
